## 최상위 목표
"현재 불안정한 YH 원격 DB(133.186.214.219:27455) 연결을 DB 설정 모델 기반의 플래그로 제어 가능하게 만들고, SQL 덤프 import를 통해 로컬 백업 DB를 구축하여 시스템이 원격 연결 없이도 안정적으로 운영되도록 한다. 이후 YH 측 접속 허용 시 원격 연결을 복구할 수 있는 구조를 만든다."

## 현재 vs 목표

| 항목 | 현재 (AS-IS) | 목표 (TO-BE) |
|------|-------------|-------------|
| 원격 DB 연결 제어 | settings.py의 DB_TYPE 환경변수로만 제어, 코드 레벨 where 분기 | DB 설정 모델로 동적 제어, 런타임에 on/off |
| 로컬 백업 DB 활용 | SQL 덤프 파일이 별도 위치에 있으나 import 로직 미구현 | SQL 덤프 import 관리 명령 및 자동화 |
| 연결 실패 시 대응 | ERPConnectionConfig의 fallback_to_mock=True로 모의 데이터 | 로컬 백업 DB에서 실제 과거 데이터 조회 |
| 원격 연결 복구 | YH 측 접속 허용 시 수동 settings.py 수정 필요 | DB 설정 모델의 플래그만으로 즉시 전환 |

## 완료·검증 기준

| 완료 조건 | 검증 방법 | 검증 주체 |
|----------|----------|----------|
| DB 설정 모델이 Django 마이그레이션으로 생성됨 | `makemigrations` + `migrate` 성공, 테이블 존재 확인 | AI |
| SQL 덤프 import 관리 명령이 덤프 파일을 읽고 import함 | 관리 명령 실행 후 DB 레코드 수 증가 확인 | AI |
| DB 설정 모델의 enabled=False로 설정 시 원격 연결 시도하지 않음 | Django admin이나 API로 플래그 변경 후 로그 확인 | AI |
| 로컬 백업 DB에서 대시보드 데이터 정상 반환 | 원격 비활성화 상태에서 대시보드 5종 curl 테스트 | AI |
| 원격 연결 복구 시 즉시 전환됨 | enabled=True로 변경 후 원격 DB 연결 성공 로그 확인 | AI |
| 프로덕션 배포 후 정상 작동 | ax.claros.co.kr에서 로컬 기반 대시보드 확인 | AI |

## 총평
DB 설정 모델(ERPConnectionConfigModel)을 생성하여 원격 DB 연결 on/off를 동적 제어하는 구조를 구현했습니다. SQL 덤프 import 관리 명령, 연결 제어 서비스 리라이트, 관리 API까지 포함하여 전체 시스템이 원격 연결 없이도 로컬 백업 DB 기반으로 운영될 수 있는 기반을 완성했습니다. 향후 실제 덤프 파일 import와 대시보드 curl 테스트, 프로덕션 배포를 통해 최종 검증이 필요합니다.
