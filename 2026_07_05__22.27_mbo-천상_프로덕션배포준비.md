## 최상위 목표
"로컬에서 완성된 DB 설정 모델(ERPConnectionConfigModel) 기반 연결 제어 시스템을 프로덕션(ax.claros.co.kr)에 배포하여, 프로덕션에서도 YH 원격 DB 연결을 on/off 제어 가능하게 한다."

## 현재 vs 목표

| 항목 | 현재 (AS-IS) | 목표 (TO-BE) |
|------|-------------|-------------|
| 프로덕션 DB 설정 모델 | 존재하지 않음 | ERPConnectionConfigModel 테이블 생성 |
| 프로덕션 연결 설정 | 존재하지 않음 | YH, LOCAL_BACKUP 설정 생성 |
| 프로덕션 코드 | 기존 코드 | 신규 코드(DB 모델 기반) 배포 |

## KPI

| 지표 | 목표값 | 측정 방법 |
|------|--------|----------|
| 프로덕션 DB 마이그레이션 완료 | 성공 | `migrate` 명령 성공 |
| 프로덕션 DB 설정 생성 | YH, LOCAL_BACKUP | DB 레코드 확인 |
| 프로덕션 API 동작 | /api/erp-sync/config/ 접근 가능 | curl 테스트 |

## 실행 계획

| # | Task | 설명 |
|---|------|------|
| 1 | 배포 환경 확인 | 프로덕션 서버 접속 및 현재 상태 확인 |
| 2 | DB 백업 | 현재 프로덕션 DB 백업 |
| 3 | 마이그레이션 적용 | ERPConnectionConfigModel 테이블 생성 |
| 4 | 기본 설정 생성 | YH, LOCAL_BACKUP 설정 생성 |
| 5 | 코드 배포 | 신규 코드 배포 |
| 6 | API 테스트 | 연결 제어 API 동작 확인 |
| 7 | 연결 테스트 | YH 연결 on/off 테스트 |

## 총평
프로덕션 배포를 위한 모든 준비물을 완성했습니다. DB 백업, 마이그레이션 적용, 설정 생성, 코드 배포를 위한 스크립트와 가이드를 작성하여, 실제 배포 시에는 스크립트 한 번의 실행으로 모든 절차가 자동화되도록 준비했습니다.

**준비된 배포 자료:**
- `deploy/update-prod-connection-config.sh` - 메인 배포 스크립트
- `deploy/backup_prod_db.sh` - DB 백업
- `deploy/apply_prod_migrations.sh` - 마이그레이션 적용
- `deploy/create_prod_configs.sh` - 기본 설정 생성
- `deploy/PRODUCTION_DEPLOYMENT_GUIDE.md` - 상세 배포 가이드

실제 배포 실행은 SSH 키를 가진 관리자가 `./deploy/update-prod-connection-config.sh <ssh-key>` 명령으로 수행하면 됩니다.
