## 1. 시스템 소개

### 1.1 개요

ERP 매핑 관리 시스템은 다양한 ERP 시스템과의 연계를 위한 테이블/필드 매핑을 UI 기반으로 관리하는 시스템입니다. 코드 수정 없이 새로운 ERP 시스템을 연결하고 매핑을 관리할 수 있습니다.

### 1.2 주요 기능

- **ERP 소스 관리**: 다중 ERP 시스템 등록 및 관리
- **테이블 매핑**: 소스 테이블 → 타겟 모델 매핑
- **필드 매핑**: 소스 필드 → 타겟 필드 매핑
- **CSV 가져오기**: 테이블 정의서 CSV 가져오기
- **동기화 모니터링**: 데이터 동기화 현황 모니터링
- **Dashboard 레이어**: 경영진단용 종합 대시보드
- **KPI 레이어**: 핵심 성과 지표 분석

### 1.3 현재 구현 현황

| 항목 | 수량 | 설명 |
|------|------|------|
| ERP 소스 | 1 | YH (유한 DB) |
| 소스 테이블 | 90 | YH ERP 테이블 |
| 타겟 모델 | 124 | 15개 앱 라벨 |
| 테이블 매핑 | 162 | 소스→타겟 연결 |
| 필드 매핑 | 1,618 | 소스 필드→타겟 필드 |

## 3. ERP 소스 관리

### 3.1 ERP 소스 목록

**메뉴**: ERP 소스 관리

화면에 등록된 모든 ERP 소스가 표시됩니다.

| 항목 | 설명 |
|------|------|
| 소스 코드 | YH, FOM, SAP 등 시스템 코드 |
| 소스명 | 시스템 이름 |
| 타입 | PostgreSQL, MS SQL, Oracle 등 |
| 상태 | 활성/비활성 상태 |
| 테이블 수 | 연결된 테이블 수 |
| 기본 여부 | 기본 소스인지 표시 |

### 3.2 ERP 소스 등록

1. [ERP 소스 관리] 페이지에서 [신규 등록] 버튼 클릭
2. 다음 정보 입력:

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 소스 코드 | 시스템 식별 코드 (영문 대문자, 20자 이내) | Y |
| 소스명 | 시스템 이름 | Y |
| 소스 타입 | 데이터베이스 유형 선택 | Y |
| 호스트 | 서버 주소 | Y* |
| 포트 | 포트 번호 | Y* |
| 데이터베이스명 | 데이터베이스 이름 | Y* |
| 설명 | 시스템 설명 | N |

* API 타입 제외

3. [저장] 버튼 클릭

**예시: YH ERP 등록**
```
소스 코드: YH
소스명: 유한 DB
소스 타입: PostgreSQL
호스트: 133.186.214.219
포트: 27455
데이터베이스: sap_yuhan
```

### 3.3 연결 테스트

1. ERP 소스 목록에서 테스트할 소스 선택
2. [연결 테스트] 버튼 클릭
3. 테스트 결과 확인:

- ✅ **성공**: 연결 정보와 지연 시간 표시
- ❌ **실패**: 오류 메시지 표시

### 3.4 테이블 가져오기

1. ERP 소스 선택
2. [테이블 가져오기] 버튼 클릭
3. 가져오기 옵션 설정:

| 옵션 | 설명 |
|------|------|
| 모듈 필터 | 특정 모듈만 가져오기 (SD, PP, QM, MM, FI, HR, CO) |
| 테이블명 패턴 | 테이블명 필터 (와일드카드 지원, 예: %_YH) |
| 필드 포함 | 테이블의 필드 정보도 함께 가져오기 |

4. [가져오기] 버튼 클릭
5. 결과 확인

### 6.2 필드 매핑 방법

### 6.3 매핑 옵션

| 옵션 | 설명 |
|------|------|
| 키 필드 | 기본키/외래키 등 키 필드 여부 |
| 필수 매핑 | 필수 매핑 여부 |
| 변환 규칙 | 데이터 변환 규칙 선택 |
| 변환 표현식 | 복잡한 변환의 경우 JSON 표현식 입력 |
| 기본값 | 소스가 NULL인 경우 사용할 기본값 |

### 6.4 변환 규칙

| 규칙 | 설명 | 사용 예시 |
|------|------|----------|
| none | 변환 없음 | 기본값 |
| upper | 대문자로 변환 | 코드 필드 |
| lower | 소문자로 변환 | |
| trim | 공백 제거 | 문자열 필드 |
| date_format | 날짜 형식 변환 | YYYYMMDD → YYYY-MM-DD |
| decimal_cast | 소수형 변환 | 문자열 → 숫자 |
| concat | 문자열 결합 | [성] + [명] → [성명] |
| lookup | 참조 테이블 조회 | 코드 → 명 |
| custom | 사용자 정의 | Python 표현식 |

## 7. CSV 가져오기/내보내기

### 7.1 CSV 형식 (테이블 정의)

```csv
테이블명,테이블 설명,모듈구분,컬렴명,컬렴설명,데이터타입,PK여부,NULL허용
SDY100_YH,영업연월별계획,SD,plan_mon,계획월,VARCHAR(6),N,N
SDY100_YH,영업연월별계획,SD,cust_nm,거래처명,VARCHAR(100),Y,Y
...
```

**주의**: CSV 파일의 컬럼명은 "컬럼명"이 아니라 **"컬렴명"**입니다.

### 7.2 CSV 가져오기 절차

1. [CSV 가져오기] 탭 선택
2. ERP 소스 선택
3. CSV 파일 선택 또는 드래그앤드롭
4. [미리보기] 버튼 클릭
5. 데이터 확인
6. [가져오기] 버튼 클릭
7. 결과 확인

### 7.3 매핑 내보내기

1. [내보내기] 탭 선택
2. 내보내기 옵션 설정:

| 옵션 | 설명 |
|------|------|
| ERP 소스 | 내보낼 소스 선택 |
| 포함할 데이터 | 테이블 정의 / 필드 정의 / 매핑 정보 |
| 형식 | CSV / JSON |

3. [내보내기] 버튼 클릭
4. 파일 다운로드

## 8. 동기화

### 8.1 동기화 타입

| 타입 | 설명 | 사용 대상 | 동기화 주기 |
|------|------|-----------|------------|
| full | 전체 데이터 동기화 | 마스터 데이터 | 초기 로드, 주간 |
| incremental | 증분 동기화 (날짜 기반) | 일일 실적 데이터 | 일일 |
| cdc | Change Data Capture | 실시간 데이터 | 실시간 (예정) |

### 8.2 동기화 우선순위

| 우선순위 | 설명 | 동기화 주기 | 예시 |
|----------|------|------------|------|
| 1 (필수) | 핵심 업무 데이터 | 실시간 ~ 1시간 | 생산실적, 출하, 회계전표 |
| 2 (중요) | 중요 운영 데이터 | 일일 | 검사결과, 재고현황 |
| 3 (일반) | 일반 참조 데이터 | 주간 | 협력사, 자재마스타 |
| 4 (확장) | 확장 분석 데이터 | 월간 | MES 태그, 이력 데이터 |

### 8.3 동기화 실행

#### 단일 테이블 동기화

1. 테이블 매핑 목록에서 동기화할 테이블 선택
2. [동기화] 버튼 클릭
3. 동기화 옵션 설정:

| 옵션 | 설명 |
|------|------|
| 동기화 타입 | 전체/증분 선택 |
| 강제 전체 | 증분 매핑이어도 전체 동기화 |
| 배치 크기 | 한 번에 처리할 레코드 수 |

4. [실행] 버튼 클릭

#### 전체 동기화

## 9. FAQ

### Q1: 현재 어떤 ERP 시스템이 연결되어 있나요?

**A**: 현재 YH (유한 DB)가 연결되어 있으며, 90개 테이블, 1,533개 필드가 정의되어 있습니다.

### Q2: Dashboard 레이어란 무엇인가요?

**A**: 경영진단용 종합 대시보드로, 8개 대시보드 모델이 제공됩니다:
- 경영진단 요약
- 영업/생산/품질/재고/구매/재무/인사 관리 대시보드

### Q3: KPI 레이어란 무엇인가요?

**A**: 핵심 성과 지표 분석용 레이어로, 9개 KPI 모델이 제공됩니다:
- 영업 실적, 추이
- 생산 실적, 설비 효율
- 품질 실적, 불량 분석
- 재고 현황, 공급업체 평가
- 재무 요약

### Q4: 새로운 확장 모델이 있나요?

**A**: 네, 14개 확장 모델이 추가되었습니다:
- 영업: 고객 불만, 영업 파이프라인
- 생산: 작업 지시, 사이클 타임 분석
- 품질: 고객 클레임
- 구매: 발주 추적, 재고 회전율
- 재무: 예산 대 실적, 부문별 수익성
- ESG: 환경 영향
- 생산성: 시간별 생산, 라인 가동률
- 개발: R&D 프로젝트
- 추적: LOT 추적

- 공통 (CO): 22개
