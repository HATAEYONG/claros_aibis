## 1. 시스템 소개

### 1.1 개요

ERP 매핑 관리 시스템은 다양한 ERP 시스템과의 연계를 위한 테이블/필드 매핑을 UI 기반으로 관리하는 시스템입니다. 코드 수정 없이 새로운 ERP 시스템을 연결하고 매핑을 관리할 수 있습니다.

### 1.2 주요 기능

- **ERP 소스 관리**: 다중 ERP 시스템 등록 및 관리
- **테이블 매핑**: 소스 테이블 → 타겟 모델 매핑
- **필드 매핑**: 소스 필드 → 타겟 필드 매핑
- **CSV 가져오기**: 테이블 정의서 CSV 가져오기
- **동기화 모니터링**: 데이터 동기화 현황 모니터링

## 2. 시작하기

### 2.1 로그인

1. 브라우저에서 시스템 URL 접속
2. 로그인 페이지에서 사용자 이름/비밀번호 입력
3. [로그인] 버튼 클릭

### 2.2 메인 화면

로그인 후 메인 대시보드가 표시됩니다. 좌측 네비게이션에서 [ERP 관리] 메뉴를 확인할 수 있습니다.

## 3. ERP 소스 관리

### 3.1 ERP 소스 목록

**메뉴**: ERP 관리 → ERP 소스 관리

화면에 등록된 모든 ERP 소스가 표시됩니다.

| 항목 | 설명 |
|------|------|
| 소스 코드 | YH, FOM, SAP 등 시스템 코드 |
| 소스명 | 시스템 이름 |
| 타입 | PostgreSQL, MS SQL, Oracle 등 |
| 상태 | 활성/비활성 상태 |
| 테이블 수 | 연결된 테이블 수 |
| 기본 여부 | 기본 소스인지 표시 |

### 3.2 ERP 소스 등록

1. [ERP 소스 관리] 페이지에서 [신규 등록] 버튼 클릭
2. 다음 정보 입력:

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 소스 코드 | 시스템 식별 코드 (영문 대문자, 20자 이내) | Y |
| 소스명 | 시스템 이름 | Y |
| 소스 타입 | 데이터베이스 유형 선택 | Y |
| 호스트 | 서버 주소 | Y* |
| 포트 | 포트 번호 | Y* |
| 데이터베이스명 | 데이터베이스 이름 | Y* |
| 설명 | 시스템 설명 | N |

* API 타입 제외

3. [저장] 버튼 클릭

### 3.3 연결 테스트

1. ERP 소스 목록에서 테스트할 소스 선택
2. [연결 테스트] 버튼 클릭
3. 테스트 결과 확인:

- ✅ **성공**: 연결 정보와 지연 시간 표시
- ❌ **실패**: 오류 메시지 표시

### 3.4 테이블 가져오기

연결 테스트 후 ERP 시스템의 테이블 정의를 가져옵니다.

1. ERP 소스 선택
2. [테이블 가져오기] 버튼 클릭
3. 가져오기 옵션 설정:

| 옵션 | 설명 |
|------|------|
| 모듈 필터 | 특정 모듈만 가져오기 |
| 테이블명 패턴 | 테이블명 필터 (와일드카드 지원) |
| 필드 포함 | 테이블의 필드 정보도 함께 가져오기 |

4. [가져오기] 버튼 클릭
5. 가져오기 결과 확인

## 4. 테이블 매핑

### 4.1 테이블 매핑 목록

**메뉴**: ERP 관리 → 테이블 매핑

등록된 모든 테이블 매핑이 표시됩니다.

| 항목 | 설명 |
|------|------|
| 매핑 코드 | 매핑 식별 코드 |
| 소스 테이블 | ERP 테이블명 |
| 타겟 모델 | MIS 모델명 |
| 동기화 우선순위 | 1(필수) ~ 4(확장) |
| 동기화 타입 | 전체/증분/CDC |
| 활성화 | 활성/비활성 상태 |
| 마지막 동기화 | 최근 동기화 시간 |

### 4.2 테이블 매핑 등록

1. [테이블 매핑] 페이지에서 [신규 매핑] 버튼 클릭
2. 매핑 정보 입력:

#### 기본 정보

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 매핑 코드 | 매핑 식별 코드 | Y |
| 매핑명 | 매핑 이름 | Y |
| ERP 소스 | 소스 ERP 선택 | Y |
| 소스 테이블 | 소스 테이블 선택 | Y |
| 타겟 모델 | 타겟 MIS 모델 선택 | Y |
| 설명 | 매핑 설명 | N |

#### 동기화 설정

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 동기화 우선순위 | 1:필수(실시간), 2:중요(시간별), 3:일반(일별), 4:확장(주별/월별) | Y |
| 동기화 타입 | full: 전체, incremental: 증분, cdc: 변경 데이터 캡처 | Y |
| 날짜 컬럼 | 증분 동기화 기준 컬럼 (incremental인 경우 필수) | Y* |
| 활성화 | 매핑 활성화 여부 | Y |

* incremental 타입인 경우 필수

3. [저장] 버튼 클릭
4. 저장 후 자동으로 필드 매핑 페이지로 이동

### 4.3 매핑 검증

매핑 유효성을 검증합니다.

1. 테이블 매핑 목록에서 검증할 매핑 선택
2. [검증] 버튼 클릭
3. 검증 결과 확인:

| 검증 항목 | 설명 |
|----------|------|
| 구조 검증 | 테이블/모델 존재, 키 필드 매핑 등 |
| 필드 검증 | 필드 타입 호환성, 필수 매핑 등 |
| 연결 검증 | 소스 테이블 연결 가능 여부 |

## 5. 필드 매핑

### 5.1 필드 매핑 화면

**접근 방법**: 테이블 매핑 상세 → [필드 매핑] 탭

화면은 3개 영역으로 구성됩니다:

1. **소스 필드**: ERP 테이블의 필드 목록
2. **타겟 필드**: MIS 모델의 필드 목록
3. **매핑 영역**: 현재 매핑된 필드 목록

### 5.2 필드 매핑 방법

#### 방법 1: 드래그앤드롭

1. 소스 필드를 드래그
2. 타겟 필드에 드롭
3. 매핑 설정 다이얼로그에서 옵션 설정
4. [확인] 버튼 클릭

#### 방법 2: 자동 매칭

1. [자동 매칭] 버튼 클릭
2. 시스템이 필드명 기반으로 자동 매칭
3. 매핑 결과 확인 후 필요시 수정

#### 방법 3: 수동 매핑

1. [매핑 추가] 버튼 클릭
2. 소스 필드 선택
3. 타겟 필드 선택
4. 매핑 옵션 설정
5. [확인] 버튼 클릭

### 5.3 매핑 옵션

| 옵션 | 설명 |
|------|------|
| 키 필드 | 기본키/외래키 등 키 필드 여부 |
| 필수 매핑 | 필수 매핑 여부 |
| 변환 규칙 | 데이터 변환 규칙 선택 |
| 변환 표현식 | 복잡한 변환의 경우 JSON 표현식 입력 |
| 기본값 | 소스가 NULL인 경우 사용할 기본값 |

### 5.4 변환 규칙

| 규칙 | 설명 | 사용 예시 |
|------|------|----------|
| 없음 | 변환 없음 | 기본값 |
| 대문자 | 대문자로 변환 | 고객코드 → CUSTOMER_CODE |
| 소문자 | 소문자로 변환 | |
| 공백 제거 | 앞뒤 공백 제거 | |
| 날짜 형식 | 날짜 형식 변환 | YYYYMMDD → YYYY-MM-DD |
| 소수형 변환 | 문자열을 숫자로 변환 | "123.45" → 123.45 |
| 문자열 결합 | 여러 필드 결합 | [성] + [명] → [성명] |
| 룩업 | 참조 테이블 조회 |
| 사용자 정의 | Python 표현식 | 복잡한 변환 로직 |

### 5.5 필드 매핑 검증

필드 매핑 완료 후 [검증] 버튼으로 유효성 확인:

- ✅ **통과**: 모든 검증 통과
- ⚠️ **경고**: 비호환 타입 등 경고
- ❌ **실패**: 필수 매핑 누락 등 오류

## 6. 가져오기/내보내기

### 6.1 CSV 테이블 정의서 가져오기

**메뉴**: ERP 관리 → 가져오기/내보내기

#### CSV 형식

```csv
테이블명,테이블 설명,모듈구분,컬럼명,컬럼설명,데이터 타입,NOT NULL,PK 여부,FK 여부
SDY100_YH,월별 판매 계획,영업,cust_cd,고객코드,varchar(20),Y,Y,N
SDY100_YH,월별 판매 계획,영업,plan_date,계획일,date,Y,N,N
```

#### 가져오기 절차

1. [CSV 가져오기] 탭 선택
2. ERP 소스 선택
3. CSV 파일 선택 또는 드래그앤드롭
4. [미리보기] 버튼 클릭
5. 데이터 확인
6. [가져오기] 버튼 클릭
7. 결과 확인

### 6.2 매핑 내보내기

1. [내보내기] 탭 선택
2. 내보내기 옵션 설정:

| 옵션 | 설명 |
|------|------|
| ERP 소스 | 내보낼 소스 선택 |
| 포함할 데이터 | 테이블 정의 / 필드 정의 / 매핑 정보 |
| 형식 | CSV / JSON |

3. [내보내기] 버튼 클릭
4. 파일 다운로드

### 6.3 템플릿 다운로드

CSV 템플릿을 다운로드하여 형식을 확인할 수 있습니다.

1. [템플릿 다운로드] 버튼 클릭
2. 템플릿 파일 다운로드

## 7. 동기화 모니터링

### 7.1 동기화 현황

**메뉴**: ERP 관리 → 동기화 모니터링

전체 동기화 현황을 한눈에 확인할 수 있습니다.

| 항목 | 설명 |
|------|------|
| 총 테이블 수 | 매핑된 테이블 총수 |
| 활성 테이블 수 | 활성화된 테이블 수 |
| 진행 중 | 현재 동기화 중인 테이블 수 |
| 성공 | 최근 24시간 성공 수 |
| 실패 | 최근 24시간 실패 수 |

### 7.2 동기화 실행

#### 단일 테이블 동기화

1. 테이블 매핑 목록에서 동기화할 테이블 선택
2. [동기화] 버튼 클릭
3. 동기화 옵션 설정:

| 옵션 | 설명 |
|------|------|
| 동기화 타입 | 전체/증분 선택 |
| 강제 전체 | 증분 매핑이어도 전체 동기화 |
| 배치 크기 | 한 번에 처리할 레코드 수 |

4. [실행] 버튼 클릭

#### 전체 동기화

1. [동기화 모니터링] 페이지에서 [전체 동기화] 버튼 클릭
2. 동기화 우선순위 선택:

| 우선순위 | 대상 |
|---------|------|
| 필수 | 우선순위 1 (실시간) |
| 높음 | 우선순위 1~2 |
| 보통 | 우선순위 1~3 |
| 전체 | 모든 우선순위 |

3. [실행] 버튼 클릭

### 7.3 동기화 로그

동기화 이력을 조회할 수 있습니다.

1. [동기화 모니터링] 페이지 → [로그] 탭
2. 필터 옵션 설정:

| 필터 | 설명 |
|------|------|
| 테이블명 | 특정 테이블만 조회 |
| 상태 | 성공/실패/진행중 |
| 기간 | 날짜 범위 |

3. [조회] 버튼 클릭
4. 로그 상세 확인

### 7.4 로그 상세

로그 항목을 클릭하면 상세 정보를 확인할 수 있습니다.

| 항목 | 설명 |
|------|------|
| 시작 시간 | 동기화 시작 시간 |
| 종료 시간 | 동기화 종료 시간 |
| 소요 시간 | 총 소요 시간 |
| 처리 레코드 수 | 전체 처리 레코드 수 |
| 성공 레코드 수 | 성공한 레코드 수 |
| 실패 레코드 수 | 실패한 레코드 수 |
| 에러 메시지 | 실패 시 에러 메시지 |

## 9. FAQ

### Q1: 새 ERP 시스템을 추가하려면 어떻게 하나요?

**A**: [ERP 소스 관리] → [신규 등록] → 정보 입력 → [연결 테스트] → [테이블 가져오기] 순서로 진행하세요.

### Q2: 기존 매핑을 복사해서 새로 만들 수 있나요?

**A**: 네, 테이블 매핑 상세에서 [복제] 버튼을 클릭하면 매핑을 복사할 수 있습니다.

### Q3: 동기화가 실패했습니다. 어떻게 하나요?

**A**: [동기화 로그]에서 실패한 동기화를 선택하여 [에러 메시지]를 확인하세요. 일반적인 오류는 연결 문제, 권한 문제, 데이터 형식 불일치 등입니다.

### Q4: 증분 동기화는 어떻게 설정하나요?

**A**: 테이블 매핑에서 [동기화 타입]을 '증분'으로 선택하고 [날짜 컬럼]을 지정하세요.

### Q5: CSV 가져오기가 실패합니다.

**A**: CSV 파일이 템플릿 형식을 따르는지 확인하세요. 첫 행은 헤더여야 하며 쉼표(,)로 구분되어야 합니다.
