## 4. 데이터 동기화 방안

### 4.1 동기화 유형

| 유형 | 대상 테이블 | 주기 | 방식 |
|------|------------|------|------|
| 실시간 | 생산실적, MES데이터 | 1분 | Trigger/API |
| 준실시간 | 입출고, 검사결과 | 5분 | Polling |
| 배치 | 마감데이터, 계획 | 일 1회 | Batch Job |
| 온디맨드 | 리포트용 집계 | 요청시 | API Call |

### 4.2 연동 아키텍처

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   EMAX ERP DB   │────▶│  Sync Service   │────▶│  MIS Dashboard  │
│   (MS-SQL)      │     │  (Django)       │     │  (PostgreSQL)   │
└─────────────────┘     └─────────────────┘     └─────────────────┘
         │                      │                       │
         │                      ▼                       │
         │              ┌───────────────┐               │
         └─────────────▶│ Staging Table │◀──────────────┘
                        │ (ETL 중간저장) │
                        └───────────────┘
```

### 4.3 공통 필드 매핑

| EMAX 공통 필드 | 설명 | MIS 매핑 필드 |
|---------------|------|--------------|
| bs_cd | 사업장코드 | business_site |
| co_cd | 회사코드 | company_code |
| fac_cd | 공장코드 | factory_code |
| wh_cd | 창고코드 | warehouse_code |
| cust_cd | 거래처코드 | customer_code / supplier_code |
| itm_id | 품목ID | product_id / material_id |
| cid | 생성자ID | created_by |
| cdt | 생성일시 | created_at |
| mid | 수정자ID | modified_by |
| mdt | 수정일시 | modified_at |

## 5. 우선순위별 연동 테이블

### Phase 1 (필수)
1. `SDY100_YH` - 매출계획 → Sales.MonthlySales
2. `SDA500_YH`, `SDA510_YH` - 출하계획 → Production.WorkOrder
3. `QMO100`, `QMO110` - 검사결과 → Quality.QualityInspection
4. `LCB100` - 재고현황 → Purchase.Inventory

### Phase 2 (중요)
5. `CAM200_YH`, `CAM300_YH` - 재무데이터 → Financial.FinancialStatement
6. `BAR200` - 입고정보 → Purchase.PurchaseOrder
7. `MRP100_yh` - 생산계획 → Production.DailyProduction
8. `QMM630`, `QMM640` - 협력사평가 → Quality.Supplier

### Phase 3 (확장)
9. `MESTagValue_YH` - MES 데이터 → Production.Equipment
10. `LET400`, `LET500` - 물류정보 → Logistics
11. `QMM300_YH`, `QPM100_YH` - SPC 데이터 → Quality.ProcessCapability
