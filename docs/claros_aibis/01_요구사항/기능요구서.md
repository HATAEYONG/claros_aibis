# 기능 요구사항

## 개요

claros_aibis 시스템의 기능적 요구사항을 상세 정의한다. 별도 문서로 분리하여 가독성을 높인다.

## 문서 개정 이력

| 버전 | 일자 | 작성자 | 수정 내용 |
|------|------|--------|----------|
| 1.0 | 2026-07-01 | AI Agent | 초기 작성 |

## 1. 마스터 데이터 관리 (P0)

### 1.1 계정과목 관리 (M-001)

**기능 개요**: 회계 처리를 위한 계정과목 코드를 체계적으로 관리한다.

**상세 기능**:
- **조회**: 계정과목 목록 조회 (코드, 명칭, 유형, 분류별 필터링)
- **생성**: 신규 계정과목 등록 (코드 중복 체크)
- **수정**: 계정과목 정보 변경 (코드 제외)
- **삭제**: 미사용 계정과목 비활성화
- **일괄 작업**: 대량 등록/수정/삭제
- **내보내기**: CSV/Excel 형식으로 계정과목 데이터 내보내기
- **가져오기**: CSV/Excel 형식으로 계정과목 데이터 가져오기

**데이터 항목**:
- 계정 코드 (필수, 고유)
- 계정과목명 (필수)
- 계정과목명(영문)
- 계정 유형 (자산/부채/자본/수익/비용)
- 대분류 (예: 자산)
- 중분류 (예: 유동자산)
- 통제 계정 (상위 계정 참조)
- 통합 항목 여부
- 세무 관련 여부
- 세목 코드
- 설명
- 활성화 여부

**UI 요구사항**:
- 계정 유형별 탭 분류 제공
- 계정과목명/코드 검색 기능
- 대분류별 필터링
- 페이징 (페이지당 20건)
- 엑셀 다운로드 (전체 또는 필터링 결과)

### 1.2 창고 관리 (M-002)

**기능 개요**: 재고 관리를 위한 창고 정보를 체계적으로 관리한다.

**상세 기능**:
- **조회**: 창고 목록 조회 (코드, 명칭, 유형, 공장별 필터링)
- **생성**: 신규 창고 등록
- **수정**: 창고 정보 변경
- **삭제**: 미사용 창고 비활성화
- **일괄 작업**: 대량 등록/수정/삭제
- **내보내기/가져오기**: CSV/Excel 형식 지원

**데이터 항목**:
- 창고 코드 (필수, 고유)
- 창고명 (필수)
- 창고명(영문)
- 창고 유형 (원자재/완제품/반제품/부품/소모품/일반)
- 공장 (예: P1)
- 건물
- 층
- 위치
- 용량
- 용량 단위
- 현재 사용률
- 관리자 (MasterEmployee 참조)
- 온도 제어 여부
- 최저/최고 온도
- 활성화 여부

### 1.3 공정 관리 (M-003)

**기능 개요**: 생산 공정 정보를 관리하고 표준 공시간을 설정한다.

- **내보내기/가져오기**: CSV/Excel 형식 지원

**데이터 항목**:
- 공정 코드 (필수, 고유)
- 공정명 (필수)
- 공정명(영문)
- 공정 유형 (주조/가공/성형/조립/검사/포장/열처리/표면처리/시험/기타)
- 공정 분류
- 공장
- 라인
- 작업장
- 표준 공시간(초)
- 표준 준비 시간(분)
- 표준 능률(개/시)
- 관련 설비 (MasterEquipment 다대다)
- 공정 담당자
- 품질 기준
- 합격 기준
- 활성화 여부

### 1.4 은행 관리 (M-004)

**기능 개요**: 금융 기관 및 결제 처리를 위한 은행 정보를 관리한다.

**상세 기능**:

- 은행명 (필수)
- 은행명(영문)

## 2. KPI 관리 (P0)

### 2.1 KPI 정의 관리 (K-001)

**기능 개요**: 8개 카테고리 80개 KPI를 정의하고 관리한다.

**상세 기능**:
- **조회**: KPI 정의 목록 조회 (카테고리별, 도메인별 필터링)
- **생성/수정/삭제**: KPI 정의 CRUD
- **카테고리 조회**: 8개 카테고리 목록 제공
- **레지스트리 조회**: KPI 레지스트리 전체 조회
- **동기화**: KPI 레지스트리로부터 정의 자동 생성/업데이트

**KPI 카테고리**:
| 카테고리 | 코드 | KPI 수 | 설명 |
|----------|------|--------|------|
| 재무 | financial | 10 | 매출총이익률, 영업이익률, ROI 등 |
| 원가 | cost | 10 | 재료비율, 노무비율, 원가절감액 등 |
| 생산 | production | 10 | 생산계획달성율, 가동률, 설비가동률 등 |
| 품질 | quality | 10 | 수율, 불량률, 공정불량률 등 |
| 영업 | sales | 10 | 매출액, 수주율, 납기준수율 등 |
| 구매 | purchase | 10 | 구매액, 자재납기준수율, 구매불량률 등 |
| 제조 | manufacturing | 10 | OEE, 성능율, MTBF 등 |
| 관리회계 | accounting | 10 | 원가차이, 물량차이, 단가차이 등 |

**데이터 항목**:
- KPI 코드 (필수, 고유)
- KPI명 (필수)
- KPI명(영문)
- KPI 유형 (efficiency/quality/cost/delivery/safety/financial/productivity)
- 도메인 (cost/financial/production/quality/sales/hr)
- 설명
- 집계 방법 (sum/avg/min/max/count)
- 단위
- 목표 방향 (높을수록좋음/낮을수록좋음/목표값근접)
- 경고 임계값
- 위험 임계값
- 계산 로직
- 원본 테이블
- 담당 부서
- 활성화 여부

### 2.2 KPI 실적 관리 (K-002)

**기능 개요**: KPI 실적 데이터를 조회하고 분석한다.

**상세 기능**:
- **조회**: KPI 실적 목록 조회 (KPI, 날짜, 차원별 필터링)
- **최신 실적 조회**: 각 KPI의 최신 실적값 조회
- **요약 통계**: 연도/분기/월별 KPI 현황 요약
- **상태 분포**: 양호/주의/위험/중립 상태 분포

**데이터 항목**:
- KPI
- 날짜
- 연도/분기/월/주
- 공장
- 라인
- 부서
- 제품 (선택)
- 공급처 (선택)
- 고객사 (선택)
- 실적값 (필수)
- 목표값
- 기준값
- 달성율
- 차이
- 차이율
- 상태 (good/warning/critical/neutral)
- 출처 시스템
- 출처 테이블

### 2.3 KPI 재계산 (K-003)

**기능 개요**: 특정 기간 및 차원에 대해 KPI를 대량 계산한다.

**상세 기능**:
- **대량 계산**: 기간, 차원 지정 후 KPI 재계산
- **KPI 선택**: 특정 KPI만 선택하여 계산 가능
- **비동기 실행**: 백그라운드로 대량 계산 수행

**입력 파라미터**:
- 시작 날짜 (필수)
- 종료 날짜 (필수)
- KPI 코드 목록 (선택)
- 공장 (선택)
- 라인 (선택)
- 부서 (선택)

### 2.4 KPI 레지스트리 동기화 (K-004)

**기능 개요**: KPI 레지스트리에 정의된 80개 KPI를 DB에 자동 생성한다.

**상세 기능**:
- **전체 생성**: 80개 KPI 정의를 한 번에 생성
- **업데이트**: 레지스트리 변경사항 반영
- **결과 보고**: 생성/업데이트 건수 보고

## 3. ERP 데이터 통합 (P1)

### 3.1 SAP 연동 (E-001)

**기능 개요**: SAP ERP 시스템과 데이터를 주고받는다.

**상세 기능**:
- **데이터 추출**: SAP ERP에서 데이터 추출
- **데이터 변환**: 표준 포맷으로 변환
- **데이터 적재**: 마스터 데이터/트랜잭션 데이터에 적재
- **동기화 스케줄**: 주기/실시간 동기화

**SAP 데이터 매핑**:
| 대상 | SAP 테이블 | 마스터 테이블 | 주기 |
|------|-------------|-------------|------|
| 제품 | ITEM | MasterProduct | 실시간 |
| 공급처 | VENDOR | MasterVendor | 일 1회 |
| 고객 | CUSTOMER | MasterCustomer | 일 1회 |

### 3.2 FOM 연동 (E-002)

**기능 개요**: FOM ERP 시스템과 데이터를 주고받는다.

**상세 기능**:
- **데이터 추출**: FOM ERP에서 데이터 추출
- **데이터 변환**: 표준 포맷으로 변환
- **데이터 적재**: 마스터 데이터/트랜잭션 데이터에 적재

**FOM 데이터 매핑**:
| 대상 | FOM 테이블 | 마스터 테이블 | 주기 |
|------|-------------|-------------|------|
| 설비 | EQUIPMENT | MasterEquipment | 실시간 |
| 공정 | PROCESS | MasterProcess | 일 1회 |

## 4. AI 분석 기능 (P2)

### 4.1 예측 분석 (A-001)

**기능 개요**: 매출/생산/품질 등 주요 지표를 예측한다.

**상세 기능**:
- **지표 선택**: 예측할 지표 선택
- **기간 설정**: 예측 기간 설정
- **모델 선택**: 예측 모델 선택
- **예측 실행**: 예측 수행 및 결과 저장
- **시각화**: 예측 결과 그래프 제공

**예측 지표**:
- 월별 매출액
- 일별 생산량
- 품질 불량률
- 재고 수준

### 4.2 인과 분석 (A-002)

**기능 개요**: 원인별 영향도를 분석하여 인과관계를 규명한다.

**상세 기능**:
- **원인 선택**: 분석할 원인 변수 선택
- **결과 선택**: 영향을 받는 결과 변수 선택
- **분석 실행**: 인과 분석 실행
- **시각화**: 인과 그래프 제공

### 4.3 이상 탐지 (A-003)

**기능 개요**: 데이터 이상을 탐지하고 알림을 발송한다.

**상세 기능**:
- **이상 탐지**: 통계적/기계학 학습 기반 이상 탐지
- **알림 발송**: 이상 발생 시 관리자에게 알림
- **이상 내역**: 이상 원인 분석
- **대응 가이드**: 대응 방안 제안

- **기능 타워**: 재무/영업/생산/구매/품질 타워

## 6. 사용자 시나리오

### 6.1 마스터 데이터 관리 시나리오

```
1. 사용자가 마스터 데이터 관리 메뉴 클릭
2. 계정과목 탭에서 전체 계정과목 목록 조회
3. 신규 계정과목 등록 버튼 클릭
4. 필수 항목(계정 코드, 명칭, 유형) 입력
5. 저장 버튼 클릭하여 등록 완료
6. 엑셀 다운로드 버튼으로 전체 계정과목 내보내기
```

### 6.2 KPI 관리 시나리오

```
1. 사용자가 KPI 관리 메뉴 클릭
2. 재무 카테고리 탭 선택
3. 10개 재무 KPI 카드 표시 확인
4. "전체 재계산" 버튼 클릭
5. 시작일/종료일 입력 후 재계산 실행
6. 재계산 완료 메시지 확인
7. 업데이트된 KPI 실적값 확인
```

## 9. 변경 관리

### 9.1 요구사항 변경 절차
1. PO 또는 이해관계자가 요구사항 변경 요청
2. 영향도 평가 (일정/비용/품질)
3. 변경 승인
4. 요구사항 문서 업데이트
5. 관련 팀 통보

### 9.2 변경 이력 관리
- 요구사항 ID별 변경 이력 추적
- 변경 전/후 비교 문서 유지
- 이해관계자 변경 통보
