## 4. 뷰 (Views)

### 4.1 v_kpi_latest (최신 KPI 실적)

```sql
CREATE VIEW v_kpi_latest AS
SELECT DISTINCT ON (kpi_id, plant, department)
    kpi_id,
    kpi_code,
    kpi_name,
    category,
    report_date,
    plant,
    department,
    actual_value,
    target_value,
    status
FROM kpi_fact
JOIN kpi_definition USING (kpi_id)
ORDER BY kpi_id, plant, department, report_date DESC;
```

### 4.2 v_kpi_summary (KPI 요약)

```sql
CREATE VIEW v_kpi_summary AS
SELECT
    category,
    COUNT(*) AS total_kpis,
    SUM(CASE WHEN status = 'good' THEN 1 ELSE 0 END) AS good_count,
    SUM(CASE WHEN status = 'warning' THEN 1 ELSE 0 END) AS warning_count,
    SUM(CASE WHEN status = 'critical' THEN 1 ELSE 0 END) AS critical_count
FROM kpi_fact
WHERE report_date = CURRENT_DATE
GROUP BY category;
```

## 6. 백업/복구 전략

### 6.1 백업 정책

| 항목 | 주기 | 방법 | 보관 기간 |
|------|------|------|-----------|
