## 1. 아키텍처 개요

### 1.1 아키텍처 스타일

**3-Tier Architecture with Microservices Pattern**

```
┌─────────────────────────────────────────────────────────┐
│                     Presentation Layer                   │
│                    (React 19 + TypeScript)                │
└────────────────────┬────────────────────────────────────┘
                     │ REST API
┌────────────────────┴────────────────────────────────────┐
│                     Application Layer                     │
│              (Django 4.x + DRF 3.x)                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │   Data   │  │   KPI    │  │   ERP    │  │   AI     │  │
│  │   Hub    │  │ Engine   │  │   Sync   │  │   Core   │  │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘  │
└────────────────────┬────────────────────────────────────┘
                     │ ORM/SQL
┌────────────────────┴────────────────────────────────────┐
│                      Data Layer                           │
│                  (PostgreSQL 14+)                        │
└─────────────────────────────────────────────────────────┘
```

### 1.2 계층별 책임

#### Presentation Layer
- 사용자 인터페이스 제공
- 사용자 입력 처리 및 검증
- 데이터 시각화
- 브라우저 상태 관리

#### Application Layer
- 비즈니스 로직 실행
- API 요청/응답 처리
- 데이터 변환 및 가공
- 외부 시스템 연동

#### Data Layer
- 데이터 영구 저장
- 트랜잭션 관리
- 데이터 무결성 보장

### 2.2 핵심 컴포넌트

#### 2.2.1 Data Hub (데이터 허브)

**Master Data Module**
- 마스터 데이터 CRUD
- 대량 작업 지원
- 내보내기/가져오기

**Analytics Module**
- KPI 정의 관리
- KPI 실적 관리
- 대량 계산 엔진

#### 2.2.2 KPI Engine (KPI 엔진)

```
┌─────────────────────────────────────┐
│         KPI Registry                │
│    (8 카테고리 80 KPI 정의)          │
└──────────────┬──────────────────────┘
               │
┌──────────────┴──────────────────────┐
│      Unified KPI Engine              │
│  ┌─────────────────────────────┐    │
│  │  - execute_calculation()     │    │
│  │  - bulk_calculate()          │    │
│  │  - calculate_by_category()   │    │
│  └─────────────────────────────┘    │
└──────────────┬──────────────────────┘
               │
┌──────────────┴──────────────────────┐
│         Calculation Result          │
│           (KPIFact)                 │
└─────────────────────────────────────┘
```

#### 2.2.3 Mixin Architecture

```python
# 공통 Mixin 계층 구조
┌───────────────────────────┐
│  ExportImportMixin        │
│  - export_csv()           │
│  - export_excel()         │
│  - export_json()          │
│  - import_data()          │
└───────────┬───────────────┘
            │ 상속
┌───────────┴───────────────┐
│ BatchOperationsMixin     │
│ - batch_create()         │
│ - batch_update()         │
│ - batch_delete()         │
└───────────┬───────────────┘
            │ 상속
┌───────────┴───────────────┐
│    ViewSet               │
│ - CRUD 기본 기능          │
└──────────────────────────┘
```

## 4. 데이터 아키텍처

### 4.1 데이터 플로우

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│   SAP   │────▶│  Data    │────▶│ KPI      │
│   ERP    │     │  Hub     │     │ Engine   │
└──────────┘     └──────────┘     └──────────┘
┌──────────┐           │                  │
│   FOM    │───────────┘                  │
│   ERP    │                              │
└──────────┘                              │
                                          │
                              ┌───────────▼──────────┐
                              │   KPIFact (Result)    │
                              └───────────┬──────────┘
                                          │
                              ┌───────────▼──────────┐
                              │   Dashboard/Report   │
                              └──────────────────────┘
```

### 4.2 데이터 통합 전략

**ERP 데이터 동기화**
- 실시간: 제품, 설비
- 일 1회: 공급처, 고객, 공정
- 비동기 큐: 대량 트랜잭션

**KPI 계산 결과**
- 주기적 자동 계산 (매일 새벽)
- 온디맨드 재계산
- 결과 캐싱 (30분)
