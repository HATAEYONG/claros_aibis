## 4. 백업 및 복구

### 4.1 백업 정책

| 항목 | 주기 | 방법 | 보관 기간 |
|------|------|------|-----------|
| 데이터베이스 | 매일 02:00 | pg_dump | 30일 |
| 정적 파일 | 주간 | tar.gz | 90일 |
| 로그 파일 | 주간 | tar.gz | 90일 |

### 4.2 백업 스크립트

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d)
BACKUP_DIR="/backup/claros_aibis/${DATE}"

# 백업 디렉터리 생성
mkdir -p ${BACKUP_DIR}

# 데이터베이스 백업
pg_dump -U claros_user -h localhost claros_aibis | gzip > ${BACKUP_DIR}/database.sql.gz

# 정적 파일 백업
tar czf ${BACKUP_DIR}/static.tar.gz /var/www/claros_aibis/backend/static/

# 로그 백업
tar czf ${BACKUP_DIR}/logs.tar.gz /var/log/claros_aibis/

# 30일 이상 백업 삭제
find /backup/claros_aibis -type d -mtime +30 -exec rm -rf {} \;

echo "Backup completed: ${BACKUP_DIR}"
```

### 4.3 복구 절차

```bash
#!/bin/bash
# restore.sh

BACKUP_DIR=$1

if [ -z "${BACKUP_DIR}" ]; then
    echo "Usage: ./restore.sh <backup_directory>"
    exit 1
fi

# 데이터베이스 복구
gunzip < ${BACKUP_DIR}/database.sql.gz | psql -U claros_user -h localhost claros_aibis

# 정적 파일 복구
tar xzf ${BACKUP_DIR}/static.tar.gz -C /

# 서비스 재시작
sudo supervisorctl restart claros_aibis
sudo systemctl reload nginx

echo "Restore completed: ${BACKUP_DIR}"
```
