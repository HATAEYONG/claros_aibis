## 2. 주요 장애 시나리오

### 2.1 서버 다운 (L1)

**증상**:
- 서비스 전면 중단
- 모든 API 502/503 응답
- 모니터링 경고 발생

**대응 절차**:
1. 서버 상태 확인 (`ssh` 접속 시도)
2. 프로세스 상태 확인 (`ps aux | grep gunicorn`)
3. 로그 확인 (`tail -f /var/log/claros_aibis/error.log`)
4. 서비스 재시작 (`sudo supervisorctl restart claros_aibis`)
5. Nginx 재시작 (`sudo systemctl restart nginx`)
6. 정상 확인 (`curl` 테스트)

**명령어**:
```bash
# 프로세스 확인
ps aux | grep gunicorn

# 서비스 재시작
sudo supervisorctl restart claros_aibis

# 로그 확인
tail -f /var/log/claros_aibis/error.log

# 정상 확인
curl http://localhost:8000/api/data-hub/master/accounts/
```

---

### 2.2 데이터베이스 연결 실패 (L1)

**증상**:
- API 500 응답
- "connection refused" 에러
- DB 연결 실패 로그

**대응 절차**:
1. DB 상태 확인 (`ping` DB 호스트)
2. DB 연결 확인 (`psql` 접속 시도)
3. 연결 풀 확인 (최대 연결 수)
4. 프로세스 재시작 (연결 초기화)
5. DB 재시작 (필요 시)

**명령어**:
```bash
# DB 접속 확인
psql -h localhost -U claros_user -d claros_aibis

# 연결 확인
SELECT count(*) FROM pg_stat_activity;

# 긴 쿼리 확인
SELECT pid, now() - query_start as duration, query
FROM pg_stat_activity
WHERE state = 'active'
ORDER BY duration DESC;

# 쿼리 취소
SELECT pg_cancel_backend(<pid>);

# DB 재시작
sudo systemctl restart postgresql
```

### 2.3 디스크 공간 부족 (L2)

**증상**:
- 파일 쓰기 실패
- "No space left on device" 에러
- 디스크 사용률 90%+ 경고

**대응 절차**:
1. 디스크 사용량 확인 (`df -h`)
2. 대용량 파일 찾기 (`du -sh *`)
3. 로그 파일 삭제 (`find` 로그 파일)
4. 임시 파일 정리 (`rm -rf /tmp/*`)
5. 백업 파일 정리 (`find` 백업 파일)

**명령어**:
```bash
# 디스크 사용량 확인
df -h

# 대용량 디렉터리 찾기
du -sh /* | sort -rh | head -10

# 30일 이상 로그 삭제
find /var/log/claros_aibis -name "*.log" -mtime +30 -delete

# 30일 이상 백업 삭제
find /backup/claros_aibis -type d -mtime +30 -exec rm -rf {} \;

# 임시 파일 정리
rm -rf /tmp/*
```

### 2.4 메모리 부족 (L2)

**증상**:
- 시스템 느려짐
- OOM (Out of Memory) 에러
- 프로세스 강제 종료

**대응 절차**:
1. 메모리 사용량 확인 (`free -h`)
2. 프로세스 메모리 확인 (`ps aux --sort=-%mem`)
3. 불필요한 프로세스 종료
4. 스왑 공간 확인
5. 서비스 재시작

**명령어**:
```bash
# 메모리 사용량 확인
free -h

# 프로세스 메모리 확인
ps aux --sort=-%mem | head -10

# 스왑 확인
swapon -s

# Django 캐시 클리어
python manage.py clear_cache
```

---

### 2.5 API 에러율 급증 (L2)

**증상**:
- API 500 응답 증가
- 에러율 5%+ 경고
- Sentry 에러 증가

**대응 절차**:
1. 에러 로그 확인 (`grep ERROR error.log`)
2. 최근 배포 내역 확인
3. 롤백 여부 결정
4. 핫픽스 적용
5. 모니터링 강화

**명령어**:
```bash
# 에러 로그 확인
grep ERROR /var/log/claros_aibis/error.log | tail -100

# 최근 배포 확인
git log --oneline -10

# 롤백
git checkout <previous-commit>
sudo supervisorctl restart claros_aibis
```

### 2.6 KPI 계산 실패 (L3)

**증상**:
- KPI 재계산 실패
- 계산 시간 초과
- 결과 저장 실패

**대응 절차**:
1. KPI 엔진 로그 확인
2. 데이터 정합성 확인
3. 계산 파라미터 확인
4. 엔진 재시작
5. 일부 재계산

**명령어**:
```bash
# KPI 로그 확인
grep KPI /var/log/claros_aibis/error.log | tail -50

# 데이터 확인
python manage.py dbshell
SELECT * FROM kpi_fact WHERE report_date = '2026-01-01';

# 재계산
python manage.py shell
>>> from erp_sync.data_hub.analytics.kpi_engine import UnifiedKPIEngine
>>> engine = UnifiedKPIEngine()
>>> engine.bulk_calculate('2026-01-01', '2026-01-31')
```

## 3. 일반 문제 해결

### 3.1 로그인 실패

**원인**:
- 비밀번호 오류
- 계정 잠금
- JWT 만료

**해결**:
```bash
# 계정 잠금 해제
python manage.py shell
>>> from django.contrib.auth import get_user_model
>>> User = get_user_model()
>>> user = User.objects.get(username='locked_user')
>>> user.is_active = True
>>> user.save()
```

---

### 3.2 느린 API 응답

**원인**:
- N+1 쿼리
- 인덱스 부족
- 캐시 미사용

**해결**:
```bash
# 쿼리 분석
python manage.py shell
>>> from django.db import connection
>>> from erp_sync.data_hub.master.models import MasterAccount
>>> queryset = MasterAccount.objects.all()
>>> print(queryset.explain())
>>> print(connection.queries)

# 인덱스 추가
python manage.py dbshell
CREATE INDEX idx_account_code ON master_account(account_code);
```

---

### 3.3 파일 업로드 실패

**원인**:
- 파일 크기 초과
- 형식 지원 안 함
- 디스크 공간 부족

**해결**:
```python
# settings.py
FILE_UPLOAD_MAX_MEMORY_SIZE = 10 * 1024 * 1024  # 10MB
DATA_UPLOAD_MAX_MEMORY_SIZE = 10 * 1024 * 1024
```

## 4. 에러 코드 매트릭

| 코드 | 설명 | 원인 | 해결 |
|------|------|------|------|
| 400 | Bad Request | 파라미터 오류 | 요청 확인 |
| 401 | Unauthorized | 인증 실패 | 토큰 확인 |
| 403 | Forbidden | 권한 없음 | 권한 확인 |

| 502 | Bad Gateway | 백엔드 다운 | 프로세스 확인 |
| 503 | Service Unavailable | 과부하 | 스케일업 |
