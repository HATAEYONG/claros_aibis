## 1. 마스터 데이터 관리 시나리오

### 1.1 계정과목 관리

#### TC-M-001: 계정과목 목록 조회

| 항목 | 내용 |
|------|------|
| ID | TC-M-001 |
| 이름 | 계정과목 목록 조회 |
| 우선순위 | P0 |
| 유형 | 단위 + 통합 |

**전제조건**:
- 로그인 상태
- 계정과목 데이터 10개 이상 존재

**테스트 단계**:
1. 계정과목 관리 페이지 접속
2. 목록 조회 확인

**예상 결과**:
- 페이지 로드 성공
- 10개 이상 계정과목 표시
- 컬럼: 선택, 코드, 명칭, 유형, 상태, 수정, 삭제

**검증 기준**:
- [ ] 목록 표시 정상
- [ ] 페이징 작동
- [ ] 정렬 가능

#### TC-M-002: 계정과목 생성

| 항목 | 내용 |
|------|------|
| ID | TC-M-002 |
| 이름 | 계정과목 생성 |
| 우선순위 | P0 |

   - 계정과목명: 테스트 계정

- 목록에 추가된 항목 표시

**검증 기준**:
- [ ] 생성 성공 (201)
- [ ] 중복 코드 시 에러 (400)
- [ ] 필수 항목 미입력 시 에러 (400)
