## 1. 테스트 전략 개요

### 1.1 테스트 철학

**테스트 피라미드**:
```
        ▲
       /E2E\          10% - 사용자 시나리오
      /------\
     / 통합  \         20% - API 통합
    /----------\
   /  단위 테스트 \     70% - 함수/메서드
  /--------------\
```

**원칙**:
1. **빠른 실패**: 단위 테스트에서 문제를 조기 발견
2. **자동화**: 모든 테스트는 CI/CD 파이프라인에서 자동 실행
3. **독립성**: 테스트 간 의존성 제거
4. **반복성**: 동일 입력에 동일 결과 보장

## 3. 통합 테스트 전략

### 3.1 API 통합 테스트

**대상**: REST API 엔드포인트

**테스트 항목**:
- CRUD 동작
- 필터링
- 검색
- 정렬
- 내보내기/가져오기
- 일괄 작업
- 권한 검증

**테스트 예시**:
```python
@pytest.mark.django_db
class TestMasterAccountAPI:
    """마스터 계정과목 API 통합 테스트"""

    def test_list_accounts(self, auth_client):
        """목록 조회"""
        response = auth_client.get('/api/data-hub/master/accounts/')
        assert response.status_code == 200
        assert 'results' in response.data

    def test_create_account(self, auth_client):
        """생성"""
        data = {
            'account_code': '110100',
            'account_name': '현금',
            'account_type': 'asset',
        }
        response = auth_client.post('/api/data-hub/master/accounts/', data)
        assert response.status_code == 201

    def test_update_account(self, auth_client, account):
        """수정"""
        data = {'account_name': '현금 및 현금성자산'}
        response = auth_client.patch(
            f'/api/data-hub/master/accounts/{account.account_id}/',
            data
        )
        assert response.status_code == 200
        assert response.data['account_name'] == '현금 및 현금성자산'

    def test_delete_account(self, auth_client, account):
        """삭제"""
        response = auth_client.delete(
            f'/api/data-hub/master/accounts/{account.account_id}/'
        )
        assert response.status_code == 204

    def test_filter_by_type(self, auth_client):
        """필터링"""
        response = auth_client.get(
            '/api/data-hub/master/accounts/',
            {'account_type': 'asset'}
        )
        assert response.status_code == 200
```

**테스트 위치**:
```

└── e2e/
    ├── master-data.spec.ts
    ├── kpi.spec.ts
    └── auth.spec.ts
```

**테스트 예시**:
```typescript
import { test, expect } from '@playwright/test';

test.describe('마스터 데이터 관리', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('http://localhost:3000');
    await page.fill('[name="username"]', 'testuser');
    await page.fill('[name="password"]', 'testpass123');
    await page.click('button[type="submit"]');
  });

  test('계정과목 목록 조회', async ({ page }) => {
    await page.click('[data-menu="masterData"]');
    await page.click('[data-tab="account"]');

    await expect(page.locator('.data-table tbody tr')).toHaveCount(
      page.locator('.data-table tbody tr').count()
    );
  });

  test('계정과목 생성', async ({ page }) => {
    await page.click('[data-menu="masterData"]');
    await page.click('[data-tab="account"]');
    await page.click('[data-action="create"]');

    await page.fill('[name="account_code"]', '999999');
    await page.fill('[name="account_name"]', '테스트 계정');
    await page.selectOption('[name="account_type"]', 'asset');
    await page.click('[data-action="save"]');

    await expect(page.locator('.toast-success')).toBeVisible();
  });
});
```

**테스트 항목**:
| 항목 | 목표 | 측정 방법 |
|------|------|----------|
| API 응답 시간 | 95th percentile 200ms 이내 | JMeter |
| 동시 사용자 | 100명 이상 | Locust |
| API TPS | 1000 TPS | JMeter |

**Locust 예시**:
```python
from locust import HttpUser, task, between

class APIUser(HttpUser):
    wait_time = between(1, 3)

    def on_start(self):
        self.client.post('/api/auth/login/', json={
            'username': 'testuser',
            'password': 'testpass123'
        })

    @task(3)
    def list_accounts(self):
        self.client.get('/api/data-hub/master/accounts/')

    @task(2)
    def list_kpis(self):
        self.client.get('/api/data-hub/analytics/definitions/')

    @task(1)
    def get_summary(self):
        self.client.get('/api/data-hub/analytics/facts/summary/')
```

## 7. CI/CD 통합

### 7.1 테스트 파이프라인

```yaml
# .github/workflows/test.yml
name: Test

on: [push, pull_request]

jobs:
  backend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: |
          pip install -r requirements/development.txt
      - name: Run tests
        run: |
          pytest --cov=erp_sync --cov-report=xml

  frontend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install dependencies
        run: npm install
      - name: Run tests
        run: npm test

  e2e-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Install dependencies
        run: npm install
      - name: Install Playwright
        run: npx playwright install
      - name: Run E2E tests
        run: npm run test:e2e
```

def sample_accounts(db):

    for i in range(10):

## 9. 테스트 실행

### 9.1 백엔드 테스트

```bash
# 전체 테스트
pytest

# 단위 테스트만
pytest tests/unit/

# 커버리지 포함
pytest --cov=erp_sync --cov-report=html

# 특정 테스트
pytest tests/unit/test_kpi_engine.py::test_calculate_kpi

# 병렬 실행
pytest -n auto
```

### 9.2 프론트엔드 테스트

```bash
# 단위 테스트
npm test

# E2E 테스트
npm run test:e2e

# 커버리지
npm test -- --coverage
```
