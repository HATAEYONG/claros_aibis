| 접속 정보 | `133.186.214.219:27455` (원격), 현재 **네트워크 접속 불가**(실측 확인) |

## 2. 연동 불가 상황 대응 설계

원격 연결이 불가능한 현재 상황을 고려해, **연결 시도 자체를 하지 않고 로컬 데이터로 완전히 대체**하는 구조로 설계했다.

```
[대시보드 API 요청]
        │
        ▼
ERPConnectionConfigModel(source_code='YH').is_enabled == False
        │
        ▼  (연결 시도 안 함)
로컬 DB 직접 조회 (Django ORM: sales.MonthlySales, production.DailyProduction 등)
        │
        ▼
값이 부족하면 timeseries_augmentation 엔진이 오늘까지 자동 연장한 데이터 사용
```

- `ERPConnectionConfigModel`이 YH를 `is_enabled=False`, `fallback_source=LOCAL_BACKUP`으로 설정.
- 대시보드 API(`erp_sync/services/dashboard_data_service.py`)는 애초에 YH 연결 코드를 호출하지 않고, Django ORM으로 로컬 모델을 직접 조회하도록 재작성됨.
- `LOCAL_BACKUP` 소스는 실제로 도달 가능한 자체 DB(로컬 SQLite 또는 운영 PostgreSQL)를 가리키며, 연결 테스트(`test-connection` API)로 실제 동작을 검증할 수 있다.

## 3. 실제 ERP 데이터 확보 시 연동 절차 (1회성 ETL)

원격 접속이 불가능하므로, 별도로 확보한 **SQL Server 백업 파일**(`erd/ERP_backup_20251117`, 7.1GB)을 로컬 Docker SQL Server 컨테이너에 복원하여 실제 이력 데이터를 추출했다.

```
1. Docker로 mcr.microsoft.com/mssql/server:2022-latest 컨테이너 기동
2. RESTORE DATABASE ERP FROM DISK = '/backup/ERP_backup_20251117'
   WITH MOVE ... (논리 파일 재배치)
3. ERPConnectionConfigModel(LOCAL_BACKUP).source_type = 'mssql'로 설정,
   실제 복원된 컨테이너를 가리키도록 host/port/database_name 갱신
4. import_real_yh_data 관리 명령: 실제 테이블(PPC100/QMM100/SDD100/SDY100/
   LCB100/MMB150/BCV100)을 조회·집계해 내부 Django 모델로 ETL
5. import_real_cost_data 관리 명령: COS520_YH(분기 누적 원가 스냅샷)를
   분기 순증분으로 환산해 손익(영업이익/순이익) 갱신
6. extend_timeseries 관리 명령: 실제 이력 이후 ~ 오늘까지 통계적으로 연장
```

## 4. 매핑 메타데이터 (ERPTableMapping/ERPFieldMapping)

| 매핑 코드 | 소스 | 타겟 | 비고 |
|---|---|---|---|
| PPC100_TO_DAILY_PRODUCTION | PPC100(생산실적) | production.DailyProduction | work_dt+wc_cd 집계 |
| QMM100_TO_QUALITY_INSPECTION | QMM100(수입검사) | quality.QualityInspection | 건별 매핑 |
| SDD100_TO_MONTHLY_SALES | SDD100(매출) | sales.MonthlySales | 연월 집계 |
| SDY100_TO_MONTHLY_SALES | SDY100(판매계획) | sales.MonthlySales | 목표 매출 |
| SDD100_TO_TOP_CUSTOMER | SDD100 + BCV100(거래처명 조인) | sales.TopCustomer | 월별 상위 5개 |
| LCB100_TO_INVENTORY | LCB100(재고보관현황) | purchase.Inventory | 품목별 집계 |
| MMB150_TO_INVENTORY | MMB150(구매발주단가) | purchase.Inventory | 재고가치 보강 |
| COS520_YH_TO_EXECUTIVE_SUMMARY | COS520_YH(원가) | reports.ExecutiveSummary | 분기 누적→월 원가 |
| SDD100_TO_EXECUTIVE_SUMMARY | SDD100 | reports.ExecutiveSummary | 매출(억원 환산) |

이 메타데이터는 `rebuild_mapping_master_data` 관리 명령으로 생성되며, 소스 컬럼 정의는 MSSQL에서 라이브 조회, 타겟 필드 정의는 Django 모델 introspection으로 채워 손으로 잘못 입력할 여지를 없앴다.

## 5. 데이터 정합성 검증 이력

과거(이번 세션 이전) `ERPTableMapping`/`ERPFieldMapping`에는 실제 존재하지 않는 앱(`dashboard`, `kpi`, `financial` 등 미설치 앱)을 가리키는 무효 레코드 172건, 필드매핑 1,629건이 있었다(설계 단계에서 상정했던 스키마와 실제 구현 스키마가 어긋난 채 방치된 것으로 추정). 전량 삭제 후 실측 기반으로 재구축했다.
