# 화면설계서

**작성일**: 2026-07-08 | **버전**: 1.0
**대상**: React SPA (`frontend/src/App.tsx` 메뉴 트리 기준, 총 147개 화면 항목)

---

## 1. 화면 구조 원칙

- **레이아웃**: 좌측 접이식 사이드바(메뉴 트리) + 상단 헤더(AI 어시스턴트 버튼, 다크모드 토글) + 우측 하단 AI 챗 패널(플로팅).
- **라우팅**: 각 메뉴 `id`가 URL 경로(`/{id}`)와 1:1 매핑된다(`App.tsx`의 `getMenuIdFromLocation`/`setActiveMenu`). 새로고침·뒤로가기 시에도 동일 화면이 복원된다.
- **반응형**: 데스크톱(≥1024px) 진입 시 사이드바 기본 열림, 그 미만에서는 기본 닫힘(`matchMedia('(min-width: 1024px)')` 감지).
- **데이터 로딩 패턴**: 화면 진입 시 `useEffect`로 관련 API를 병렬 호출(`Promise.all`) → 로딩/에러/정상 3단 상태 처리 → KPI 카드/차트/테이블 렌더링.

## 3. 핵심 화면 상세

### 3.1 통합 대시보드 (`dashboard` → `Dashboard.tsx`)

**호출 API**: `GET /api/erp-sync/dashboard/executive-summary/`(최근 3개월), `.../production/`, `.../quality/`, `.../inventory/`

| 영역 | 구성 |
|---|---|
| KPI 카드 4종 | 매출 달성률(목표/실적/전월대비 증감), 생산 달성률(계획/실적), 종합 수율(목표 95%), 재고 회전율(목표 8) |
| 비즈니스 프로세스 요약 | O2C/P2P 단계별 소요시간 탭 |
| 차트 2종(상단) | 월별 매출 추이(최근 3개월 목표/실적 막대), 재고 현황(등급 A/B/C 재고가치) |
| 차트 2종(하단) | 공정별(검사유형별) 품질 수율/FPY, Worst 5 부진재고 리스트 |
| KPI 테이블 | 부문별(영업/생산관리/품질/구매) 핵심성과지표 4행 |

### 3.2 KPI 관리 (`kpiManagement` → `KPIManagementPage.tsx`)

**호출 API**: `GET /api/data-hub/analytics/definitions/`, `.../facts/latest/`, `.../facts/summary/`

| 영역 | 구성 |
|---|---|
| 상단 요약 | 총 팩트 수, 양호/주의/위험 건수 |
| 카테고리 탭 | 재무/원가/생산/품질/영업/구매/제조/관리회계 (8개) |
| 액션 | 레지스트리 동기화, 선택 재계산, 전체 재계산 |
| KPI 카드 그리드 | 카테고리별 KPI 정의 목록(정의 없으면 "KPI 정의가 없습니다" 안내) |

### 3.3 프로세스 연관성 추적 (`processChainView` → `ProcessChainView.tsx`)

**호출 API**: `GET /api/data-hub/integration/process-chain/{product_code}/`

품목코드 선택 시 수주(IntegratedSalesOrder) → 생산(IntegratedProductionOrder) → 품질검사(IntegratedQualityRecord) → 재고(IntegratedMaterial) 4단계를 마스터데이터 FK로 연결해 하나의 흐름으로 시각화한다.

## 4. 공통 컴포넌트

| 컴포넌트 | 용도 |
|---|---|
| `KPICard` | 제목/값/부제/증감률/아이콘/색상 카드 |
| `ChartComponent` | Chart.js 래퍼(bar/line 등) |
| `LoadingState` / `ErrorState` | 로딩/에러 공통 화면 |
| `ErrorBoundary` | 컴포넌트 크래시 격리(에러 발생 시 전체 앱이 아닌 해당 트리만 재시도) |
