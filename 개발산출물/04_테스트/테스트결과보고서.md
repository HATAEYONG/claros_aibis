## 2. 상세 결과

### TC-DASH — 전체 PASS
5개 API 모두 `APIRequestFactory`로 직접 호출해 200 확인. 서로 다른 날짜(예: 2026-06-01 vs 2026-07-07)로 `sales` API를 호출해 `monthly_sales` 값이 다름을 실측 확인(정적 응답 아님). 브라우저에서 `/dashboard` 접속 후 스크린샷으로 4개 KPI 카드(매출 111.1%, 생산 129.1%, 수율 100%, 재고회전율 4) 실측값 렌더링 확인.

### TC-KPI — 5 PASS, 1건 결함 발견 후 수정
- TC-KPI-01~04: `curl`로 엔드포인트 존재 확인(403 = 인증 필요, 404 아님으로 엔드포인트 실재 확인). 인증 토큰을 이용한 200 응답까지는 이번 세션에서 검증하지 않음(자격증명을 로그에 남기지 않기 위한 보안 정책상 토큰을 출력하지 않았음).
- **TC-KPI-05**: 최초 실행 시 `TypeError: Cannot read properties of undefined (reading 'good')`로 `KPIManagementPage` 전체가 크래시(원인: `api.get()`이 axios 스타일 `{data:T}`를 반환한다고 잘못 가정한 `kpiService.ts`의 전면적 계약 불일치, 그리고 인증 실패 시 목업 폴백 데이터가 `KPISummary` 형태와 달라 `status_distribution`이 없었음). `api.ts`/`kpiService.ts`/`KPIManagementPage.tsx` 3개 파일 수정 후 재검증 — 크래시 없이 "KPI 정의가 없습니다" 등 안전한 빈 상태로 렌더링됨을 확인. **PASS(수정 후)**.
- TC-KPI-06: 미실행(카테고리 탭 순차 클릭 UI 테스트는 이번 세션 범위 밖).
