## 1. 시스템 기동/중지

### 로컬 개발환경
```bash
# 백엔드
cd backend && venv/Scripts/python.exe manage.py runserver

# 프론트엔드
cd frontend && npm run dev
```
`.claude/launch.json`에 `backend`/`frontend` 설정이 등록되어 있어 프리뷰 도구로도 기동 가능.

### 운영환경(Docker)
```bash
cd /home/ubuntu/netplus-mis
docker compose -f docker-compose.lightsail.yml up -d      # 전체 기동
docker compose -f docker-compose.lightsail.yml stop       # 전체 중지
docker compose -f docker-compose.lightsail.yml restart backend  # 개별 재기동
```

## 3. 자주 필요한 관리 명령

| 명령 | 용도 |
|---|---|
| `python manage.py extend_timeseries` | 시계열 데이터를 오늘 날짜까지 자동 연장 |
| `python manage.py extend_timeseries --dry-run` | 실제 반영 없이 예상 생성 건수만 확인 |
| `python manage.py load_sample_data` | 기본 샘플 데이터 일괄 적재(신규 환경 초기화 시) |
| `python manage.py rebuild_mapping_master_data` | ERP 매핑 메타데이터 재구축(MSSQL 원본 접근 가능할 때만) |
| `python manage.py check` | Django 시스템 점검 |
| `python manage.py migrate` | DB 마이그레이션 적용 |

## 4. ERP 연결 상태 관리

```bash
docker compose exec backend python manage.py shell -c "
from erp_sync.services.erp_connection_service import ERPConnectionService
svc = ERPConnectionService()
print('YH enabled:', svc.is_connection_enabled('YH'))
print('LOCAL_BACKUP test:', svc.test_connection('LOCAL_BACKUP'))
"
```

연결 on/off는 API로도 가능하다: `POST /api/erp-sync/connection-config/toggle-connection/ {"source_code": "YH"}`

## 5. DB 백업/복원

```bash
# 백업
./deploy/backup_prod_db.sh
# → /home/ubuntu/db_backups/netplus_mis_backup_YYYYMMDD_HHMMSS.sql

# 복원
docker exec -i netplus-postgres psql -U netplus_user -d netplus_mis < backup.sql
```
