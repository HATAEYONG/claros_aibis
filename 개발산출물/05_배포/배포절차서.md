# 배포절차서

**작성일**: 2026-07-08 | **버전**: 1.0
**적용 대상**: AWS Lightsail 운영 서버(3.38.107.23, ax.claros.co.kr)

---

## 1. 사전 준비

```bash
# 1) 로컬에서 변경사항 커밋 및 푸시
git add <변경파일>
git commit -m "설명"
git push origin main

# 2) SSH 접속 확인
ssh -i LightsailDefaultKey-ap-northeast-2.pem ubuntu@3.38.107.23
```

## 2. 표준 배포 절차

```bash
cd /home/ubuntu/netplus-mis

# 1) 최신 코드 반영
git pull origin main

# 2) 이미지 재빌드 (변경된 서비스만)
docker compose -f docker-compose.lightsail.yml build backend
docker compose -f docker-compose.lightsail.yml build frontend   # 프론트 변경 시

# 3) 컨테이너 재기동
docker compose -f docker-compose.lightsail.yml up -d backend
docker compose -f docker-compose.lightsail.yml up -d frontend   # 프론트 변경 시

# 4) 컨테이너 안정성 확인 (재시작 루프 여부)
docker ps --format '{{.Names}}\t{{.Status}}'

# 5) 마이그레이션 필요 시
docker compose -f docker-compose.lightsail.yml exec backend python manage.py migrate

# 6) 배포 후 스모크 테스트
curl -s -o /dev/null -w "%{http_code}\n" https://ax.claros.co.kr/
curl -s https://ax.claros.co.kr/api/erp-sync/dashboard/executive-summary/?period_value=$(date +%Y-%m)
```

## 3. 대량 데이터 반영이 필요한 배포(예: 실 데이터 ETL)

```bash
# 1) 운영 DB 백업 (필수)
chmod +x deploy/backup_prod_db.sh
./deploy/backup_prod_db.sh
# 결과: /home/ubuntu/db_backups/netplus_mis_backup_YYYYMMDD_HHMMSS.sql

# 2) 로컬에서 dumpdata (UTF-8 강제, Windows 콘솔 인코딩 이슈 주의)
PYTHONUTF8=1 python manage.py dumpdata <app.Model> ... --indent 2 -o dump.json

# 3) 운영 서버로 전송
scp -i <key.pem> dump.json ubuntu@3.38.107.23:/home/ubuntu/netplus-mis/
ssh ... "docker cp dump.json claros-backend:/app/dump.json"

# 4) (기존 데이터를 완전 교체하는 경우만) 대상 테이블 삭제 — 반드시 사전 PO 승인
docker compose exec backend python manage.py shell -c "Model.objects.all().delete()"

# 5) loaddata
docker compose exec backend python manage.py loaddata dump.json

# 6) 정리
ssh ... "rm -f dump.json && docker exec claros-backend rm -f /app/dump.json"
```

- [ ] 대시보드 API 5종 200 응답 + 오늘 날짜 기준 값 확인

- [ ] 컨테이너 메모리 사용량이 비정상적으로 증가하지 않는지 수 분간 관찰

## 6. 트러블슈팅 이력(참고)

| 증상 | 원인 | 조치 |
|---|---|---|
| 백엔드 컨테이너 재시작 반복(OOM) | Gunicorn worker 5개가 각각 22개 AI 에이전트 프레임워크 전체 로드 + 메모리 1.9GB 제약 | `GUNICORN_WORKERS=2` 환경변수로 축소 |
| `extension "vector" is not available` | 기본 `postgres:15-alpine` 이미지에 pgvector 확장 없음 | `pgvector/pgvector:pg15` 이미지로 전환(볼륨은 유지) |
| Django 개발서버 재시작 직후 프론트 요청 500 | 파일 변경 감지 → autoreload 재시작 중 요청 유입 | 재시작 완료(로그의 "Agent System initialization complete") 대기 후 재시도 |
| 신규 dumpdata 파일 loaddata 시 `UnicodeDecodeError` | Windows 콘솔 기본 인코딩(cp949)으로 파일 저장됨 | `PYTHONUTF8=1` 환경변수로 dumpdata 재실행 |
