# 요구사항정의서

**프로젝트명**: Claros MIS-AI (제조업 통합 경영정보시스템)
**작성일**: 2026-07-08
**버전**: 1.0

---

## 1. 개요

### 1.1 목적
유한산업(제조업)의 생산·품질·영업·구매·재무·회계·ESG 전 영역 데이터를 하나의 통합 대시보드에서 조회·분석하고, AI 어시스턴트를 통해 자연어 질의·원인분석·의사결정 지원을 제공하는 경영정보시스템(MIS)을 구축한다.

### 1.2 배경
- 기존 YH 사내 ERP(SQL Server, 원격지 운영) 데이터를 실시간으로 조회할 수 없는 구조적 제약(방화벽·접속 불가)이 있어, 별도의 통합 MIS 계층에서 데이터를 가공·저장하고 대시보드로 제공할 필요가 있다.
- 부서별(생산/품질/영업/구매/재무)로 흩어져 있던 KPI를 8개 카테고리 80개 지표로 표준화하여 한 화면에서 관리한다.

### 1.3 용어 정의
| 용어 | 설명 |
|------|------|
| MIS | Management Information System, 경영정보시스템 |
| KPI/KRI | 핵심성과지표 / 핵심위험지표 |
| 6M | Man, Machine, Material, Method, Measurement, Mother-nature — 제조 원인분석 프레임워크 |
| 4M2E | 6M에 Energy, Environment를 더한 확장 프레임워크 |
| O2C | Order to Cash — 수주-생산-출하-수금 프로세스 |
| P2P | Procure to Pay — 구매요청-발주-입고-지급 프로세스 |
| ERP 동기화(erp_sync) | 외부 ERP(YH) 테이블을 내부 모델로 매핑·적재하는 서브시스템 |

## 3. 비기능 요구사항

| ID | 분류 | 요구사항 |
|----|------|----------|
| NFR-01 | 성능 | 대시보드 API 응답은 일반적인 조회 조건에서 수 초 이내에 완료되어야 한다 |
| NFR-02 | 가용성 | 외부 ERP(YH) 연결 불가 상태에서도 전체 대시보드가 200 정상 응답을 반환해야 한다(폴백 데이터) |
| NFR-03 | 보안 | 연결 설정(DB 접속정보)의 비밀번호는 API 응답에 노출되지 않아야 한다(write-only) |
| NFR-04 | 보안 | 프로덕션 환경변수·자격증명은 코드/커밋에 평문으로 포함되지 않아야 한다 |
| NFR-05 | 이식성 | 로컬(SQLite/Windows)과 운영(PostgreSQL/Docker/Linux) 양쪽에서 동일한 코드베이스로 동작해야 한다 |
| NFR-06 | 운영 안정성 | 운영 서버(메모리 제약 환경)에서 백엔드 프로세스가 OOM으로 재시작 루프에 빠지지 않아야 한다 |
| NFR-07 | 유지보수성 | 시계열 데이터는 별도 배치 없이 관리 명령(`extend_timeseries`) 1회 실행으로 오늘 날짜까지 자동 연장되어야 한다 |
| NFR-08 | 데이터 정합성 | 통계적으로 생성된 값이라도 과거 실측 패턴(추세·변동폭)에서 벗어나지 않아야 하며, 장기 추정 시 0으로 수렴하는 등 비현실적 값을 만들지 않아야 한다 |

## 4. 제약사항

- 외부 YH ERP DB는 **읽기 전용**이며 현재 원격 접속이 불가능하다 — 어떤 기능도 YH 원격 DB에 쓰기를 시도해서는 안 된다.
- 운영 서버(AWS Lightsail)는 메모리 1.9GB로 제한적이다 — Gunicorn worker 수, 컨테이너 구성이 이 제약을 고려해야 한다.
- 프론트엔드는 React 19 + TypeScript, 백엔드는 Django 5.0 + DRF로 이미 확정되어 있다(기술 스택 변경 요구사항 없음).
