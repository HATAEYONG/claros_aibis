# 개발산출물 목록 (Software Development Life Cycle Deliverables)

**프로젝트명**: Claros MIS-AI — 제조업 통합 경영정보시스템(MIS) with AI 어시스턴트
**적용 방법론**: 폭포수(Waterfall) 기반 단계별 산출물 + 반복적 기능 확장(Iterative Enhancement)
**문서 작성일**: 2026-07-08
**대상 시스템**: 로컬 개발환경(localhost) + 운영환경(https://ax.claros.co.kr, AWS Lightsail)

---

## 산출물 체계

본 폴더는 소프트웨어 개발 생명주기(SDLC)의 각 단계 — **요구사항정의 → 분석/설계 → 구현 → 테스트 → 배포 → 유지보수** — 에 따라 실제 시스템(Claros MIS-AI)을 기준으로 작성한 산출물을 담는다. 이미 구축·운영 중인 시스템이므로, 산출물은 신규 개발 계획서가 아니라 **현재 시스템을 정확히 기술한 현황 기반(as-built) 문서**로 작성했다.

| 단계 | 폴더 | 주요 산출물 |
|------|------|------------|
| 요구사항분석 | `01_요구사항분석/` | 요구사항정의서, 요구사항추적표 |
| 분석/설계 | `02_설계/` | 시스템아키텍처설계서, DB설계서, 화면설계서, API설계서, 인터페이스설계서 |
| 구현 | `03_구현/` | 프로그램목록, 코딩표준및컨벤션 |
| 테스트 | `04_테스트/` | 테스트계획서, 테스트케이스, 테스트결과보고서 |
| 배포 | `05_배포/` | 배포계획서, 배포절차서 |
| 유지보수 | `06_유지보수/` | 유지보수계획서, 운영매뉴얼, 장애대응매뉴얼 |

## 문서 간 추적 관계

```
요구사항정의서 (01)
    │  각 요구사항 ID(REQ-xxx)
    ▼
요구사항추적표 (01) ── 설계서/구현/테스트 매핑
    │
    ├──▶ 시스템아키텍처설계서 / DB설계서 / 화면설계서 / API설계서 (02)
    │        │
    │        ▼
    │    프로그램목록 (03) — 실제 구현된 Django 앱 39개 + React 컴포넌트
    │        │
    │        ▼
    │    테스트케이스 (04) — 요구사항별 검증
    │        │
    │        ▼
    └──▶ 배포절차서 (05) → 운영매뉴얼/유지보수계획서 (06)
```

## 참고 — 기존 프로젝트 문서와의 관계

프로젝트 루트에는 이미 기능별 구현 보고서(`*_IMPLEMENTATION.md`), 기술문서(`TECHNICAL*.md`), API 문서(`API.md`), DB 스키마 문서(`DATABASE_SCHEMA.md`) 등이 다수 존재한다. 본 `개발산출물/` 폴더는 이들을 대체하지 않고, **SDLC 단계별 표준 산출물 형식으로 재구성·정리**한 것이다. 상세 구현 이력이 필요하면 프로젝트 루트의 해당 문서를 참조한다.
