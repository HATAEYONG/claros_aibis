## 예상치 못했던 발견 (추가 조치)

작업 중 **프로덕션의 `reports`/`sales`/`production`/`quality`/`purchase` 등 핵심 시계열 테이블이 전부 0건**이라는 사실을 발견했다 — 로컬 sqlite에는 있던 과거 이력 데이터가 애초에 프로덕션 Postgres로 반영된 적이 없었던 것(로컬 전용 `db.sqlite3`는 git 추적 대상이 아니었음). 이 상태로 증강 엔진만 돌리면 "증강할 이력 자체가 없어 전부 건너뜀"이 되어 목표를 달성할 수 없었다. 이미 존재하던 `python manage.py load_sample_data` 명령(오늘 기준 최근 12개월치 데이터를 생성)을 프로덕션에서 먼저 실행해 기반 이력을 만든 뒤 `extend_timeseries`를 적용하는 것으로 해결했다.

## 총평
YH 원격 DB 연결 시도를 대시보드 코드에서 완전히 제거하고, 로컬 통계 기반 시계열 증강 엔진으로 대체했다. 로컬에서 검증을 마친 뒤 프로덕션에 배포하는 과정에서 프로덕션 DB 자체에 기반 이력 데이터가 없었다는 더 근본적인 문제를 추가로 발견·해결했으며, 최종적으로 프로덕션 공개 URL(`ax.claros.co.kr`)에서 5개 대시보드 엔드포인트 모두 실제 증강 데이터로 정상 응답하는 것을 확인했다. 재발 방지를 위한 Celery Beat 일일 작업도 등록했으나, Celery worker/beat 프로세스 자체의 상시 구동은 서버 메모리 제약으로 별도 과제로 남는다.
